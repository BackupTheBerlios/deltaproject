{\bf Paquetage Identification}
	\begin{itemize}
	\item Enseignant
		\begin{itemize}
		\item Se loger :
			\begin{itemize}
			\item Pré-requis : Avoir un compte et un mot de passe
		\item Description : L'utilisateur clique sur le lien {\it S'identifier}.\\
			Apparait une boîte de dialogue. \\
			Il entre le login et le mot de passe.\\
			Il valide.
			\item Post-requis : L'utilisateur est identifié dans l'intranet.
			\end{itemize}
		\item Se deloger :
			\begin{itemize}
			\item Pré-requis : Etre identifié/logé. 
			\item Description : L'utilisateur clique sur {\it Quitter}.\\
			Apparait un message de confirmation.\\
			Il confirme.\\
			\item Post-requis : L'utilisateur se trouve dans la page d'accueil de l'intranet.
			\end{itemize}
		\item Modifier Mot de Passe :
			\begin{itemize}
			\item Pré-requis : Etre identifié/logé. 
			
			\item Description : L'utilisateur clique sur {\it Modifier ses paramètres}.\\
			Il est redirigé vers une page de paramètres personnels.\\
			Il entre son ancien mot de passe et confirme deux fois le nouveau. 
			
			\item Post-requis : Mot de passe modifié.
			\end{itemize}
		\end{itemize}
	\item Administrateur
		\begin{itemize}
		\item Ajouter compte :
			\begin{itemize}
			\item Pré-requis : Etre identifié/logé. 
			\item Description : L'administrateur clique sur le lien {\it Ajout de compte}.\\
			Il entre une page d'ajout de compte.\\
			Il ajoute un compte.\\
			Il ajoute un mot de passe qu'il confirme une seconde fois.\\
			Il aura la possibilité de générer un login et mot de passe automatiquement (le login est généré à partir du nom et prénom de l'utilisateur. Le mot de passe est généré aléatoirement).
			
			\item Post-requis : Nouveau compte ajouté dans la base de données.
			\end{itemize}

		\item Modifier compte :
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié. 
			\item Description : L'administrateur clique sur le lien {\it Modifier compte}.\\
			Il entre dans la page de modification de compte.\\
			Il recherche le compte soit par nom soit par login.\\ 
			Il a la possibilité de modifier les paramètres du compte.\\
			\item Post-requis : Le compte est modifié dans la base. 
			\end{itemize}

		\item Consulter compte :
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié. 
			\item Description : L'administrateur clique sur le lien {\it Consulter comptes}.\\
			Il entre dans la page de consultation des comptes.\\
			Il recherche le compte soit par nom soit par login.\\ 
			Il visualise le compte sélectionné.	
			\item Post-requis : Il obtient la page de consultation.\\
			La base de donnée reste inchangé.
			\end{itemize}

		\item Supprimer compte :
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié.
			\item Description : L'administrateur clique sur le lien {\it Supprimer comptes}.\\
			Il entre dans la page de suppression de comptes.\\
			Il recherche le compte soit par nom soit par login.\\
			Il sélectionne un compte.\\
			Il valide sa suppression. 
			\item Post-requis : Suppression d'un compte dans la base de donnée
			\end{itemize}
		\end{itemize}
	\end{itemize}

{\bf Paquetage Gestion Projet}
	\begin{itemize}
	\item Consultant
		\begin{itemize}
		\item Visualiser Projet :
			\begin{itemize}
			\item Pré-requis :Il faut qu'il y ait au moins un TD de créé	
			Il sélectionne successivement les liens lui permettant d'atteindre le projet.\\
			Il clique pour le visualiser.
			\item Post-requis :Le projet sélectionné est affiché au format HTML.
			\end{itemize}
		\end{itemize}
	\end{itemize}

	\begin{itemize}
	\item Enseignant :
		\begin{itemize}
		\item : Création Projet :
			\begin{itemize}
			\item Pré-requis : Etre identifié.\\
			Il faut qu'au moins un exercice de type projet (gros exercice) soit enregistré.
			\item : Description : L'enseignant pénêtre dans l'intranet.\\
			Il s'identifie avec son login et son mot de passe.\\
			Il sélectionne le lien lui permettant de se diriger vers la page de création de projet.\\
			Il sélection la section et la matière pour lesquels le projet est destiné.\\
			Ensuite, soit il sélectionne dans une liste l'exercice de type projet pré-enregistré souhaité pour ce projet ou soit l'"updload" à partir d'un fichier XML.\\
			Il valide sa sélection.
			\item Post-requis : Un nouveau projet est enregistré dans la liste des projets disponibles.
			\end{itemize}

		\item : Modification Projet :
			\begin{itemize}
			\item Pré-requis : Etre identifié.\\
			Il faut qu'au moins un projet soit enregistré.
			\item Description : L'enseignant pénêtre dans l'intranet. Il s'identifie avec son login et son mot de passe.\\
			Il sélectionne le lien lui permettant de se diriger vers la page des projets dans une section et matière précise.\\
			Il peut déplacer le projet dans une autre section et matière. \\
			Il peut modifier le projet par un autre exercice de type projet dans la liste des exercices.\\
			Il valide sa selection.
			\item : Avertissement : Si l'enseignant qui modifie le projet n'est pas son créateur, alors une sauvegarde de l'élément modifié sera effectué pour avoir la possibilité de revenir sur cette modification.
			\item Post-requis : Le projet est modifié dans la liste des projets disponibles.
			\end{itemize}

		\item : Suppression Projet :
			\begin{itemize}
			\item Pré-requis : Etre identifié.\\
			Il faut qu'au moins un projet soit enregistré.
			\item : Description : L'enseignant pénêtre dans l'intranet.\\
			Il s'identifie avec son login et son mot de passe.\\
			Il sélectionne successivement les liens lui permettant de se diriger vers la page de modification de projet.\\
			Il peut demander la suppression du projet. Il valide sa demande.
			\item : Avertissement : Un enseignant ne peut pas supprimer un projet ne lui appartenant pas.
			\item : Post-requis : le projet est supprimé (s'il est son créateur).
			\end{itemize}
		\end{itemize}
	\end{itemize}

			\begin{itemize}
			\item Pré-requis :Il faut qu'il y ait au moins un TD de créé

			Il sélectionne successivement les liens lui permettant d'atteindre le TD.\\

			\item Post-requis :Le TD sélection est telechargé au format PDF.

			\item Description : L'enseignant pénêtre dans l'intranet.\\
			Il s'identifie avec son login et son mot de passe. 
			Il sélectionne successivement les liens lui permettant de se diriger vers la page de création de TD.\\
			Il sélection la section et la matière pour lesquels le TD est destiné. Il sélectionne dans une liste les exercices pré-enregistrés souhaités pour ce TD ou par mots-clés.\\

			\item Post-requis :Un nouveau TD est enregistré dans la liste des TD disponibles dans une section et matière spécifique.\\

			\begin{itemize}
			\item Pré-requis :Etre identifié.\\
			Il faut qu'au moins un TD soit enregistré.\\
			\item Description : L'enseignant pénêtre dans l'intranet.\\ 
			Il s'identifie avec son login et son mot de passe. \\
			Il sélectionne successivement les liens lui permettant d'atteindre le TD.\\
			Il peut modifier la section et la matière pour lesquels le TD est destiné. \\
			Il peut modifier la liste des exercices du TD en supprimant certain exercice et/ou en ajoutant de nouveau exercice.\\
			Il valide la selection.\\
			\item Avertissement : Si l'enseignant qui modifie le TD n'est pas son créateur, alors une sauvegarde de l'élement modifié sera effectué pour avoir la possibilité de revenir sur cette modification.\\
			\item Post-requis : Le TD est modifié dans la liste des TD disponibles.
			\end{itemize}
		\item Supprimer Projet :

			\item Pré-requis :Etre identifié.\\
			Il faut qu'au moins un TD soit enregistré.
			\item Description : L'enseignant pénêtre dans l'intranet. \\
			Il s'identifie avec son login et son mot de passe. \\
			Il sélectionne successivement les liens lui permettant d'atteindre le TD.\\

			Il valide sa demande.

{\bf Paquetage Gestion Enseignement}
	\begin{itemize}
	\item Enseignant
		\begin{itemize}
		\item Créer Enseignement :
			\begin{itemize}
			\item Pré-requis : Etre logé 
			\item Description : L'enseignant pénêtre dans l'intranet. \\
			Il s'identifie avec son login et son mot de passe. \\
			Il sélectionne successivement les liens lui permettant de se diriger vers la page de création de section et de matière.\\
			Il désigne le nom de la nouvelle section ou matière qu'il veut créer. Il designe son emplacement d'indexation. Il valide ses choix.
			\item Post-requis : Une nouvelle section ou matière est créée.
			\end{itemize}
		\item Modifier Enseignement :
			\begin{itemize}
			\item Pré-requis : Etre logé et propriétaire du projet
			
			\item Description : L'enseignant pénêtre dans l'intranet. Il s'identifie avec son login et son mot de passe.\\
			il sélectionne successivement les liens lui permettant de se diriger vers la page de modification de section et de matière.\\
			Il désigne le nom de la section ou de la matière qu'il veut modifier. Il indique la modification à apporter. Il valide ses choix.\\
			\item : Si l'enseignant qui modifie la section ou la matière n'est pas son créateur, alors une sauvegarde de la modification sera effectué pour avoir la possibilité de revenir sur cette modification.
			\item Post-requis : une section ou matière est modifiée.
			\end{itemize}
		\item Supprimer Enseignement :
			\begin{itemize}
			\item Pré-requis : Etre logé et propriétaire du projet
			
			\item Description : L'enseignant pénêtre dans l'intranet. \\
			Il s'identifie avec son login et son mot de passe. \\
			il sélectionne successivement les liens lui permettant de se diriger vers la page de modification de section et de matière.\\
			Il selectionne la section ou la matière qu'il veut supprimer. Il valide son choix.
			\item Avertissement : Si l'enseignant qui supprimme la section ou la matière n'est pas son créateur, alors une sauvegarde celle-ci sera effectué pour avoir la possibilité de revenir en arrière.
			\item Post-requis : Une section ou matière est supprimée. 
			\end{itemize}
		\end{itemize}
	\end{itemize}

\section*{ Glossaire}

{\bf Administrateur} : Personne qui gère tous les comptes des enseigments.\\
{\bf EnseignantAdmin}: Enseignant qui rajoute et supprime des enseignements.\\
{\bf Enseignant}: Personne qui gère les exercices et les projets.\\
{\bf Consultant}: Personne qui visualise ou upload les TDs ou projet.\\
{\bf Exercice } il est formé d'un seul exercice.\\
{\bf TD}: il est formé d'un ensemble d'éxercices.\\
{\bf Projet}: il est formé d'un seul exercice.(souvent assez gros).\\
{\bf Enseignement}: un nouvel enseignement est constitué de la matière, l'année et la section.\\
