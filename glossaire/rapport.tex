\subsection{PARTIE 1:Sujet du projet}

Le but de ce projet est de réaliser une application Web permettant à
des enseignants de se constituer une base d'exercices au format XML.

Le projet devra permettre aux enseignants de rentrer des exercices au
format XML, d'éditer à partir des exercices une feuille de TD auformat
PDF.
Le TD correspondant sera aussitôt disponible au format HTML pour les
étudiants.
Le projet est une application Web comportant trois zones distinctes:

\begin{itemize}
\item Une zone de visualisation où n'importe quel étudiant peut
consulter les TDs propres à un enseignement au format HTML et PDF.
\item Une zone d'édition qui permet à n'importe quel enseignant
d'ajouter de nouveaux exercices à la base, de créer de nouveaux Tds à
partir d'exercices, et enfin, de visualiser les TDs obtenus.
\item Une zone d'administration qui permet de changer \ mettre à jour
les mots de passe et d'effectuer la configuration.
\end{itemize}



\subsection{PARTIE 2: Objectifs à atteindre}

On veut aller plus loin en proposant à nos clients comme à nos
collaborateurs un engagement permanent sur la qualité.
Les objectifs de cette charte sont de: 
\begin{itemize}
\item Fournir une entière satisfaction et une véritable autonomie à
nos clients par rapport aux solutions livrées.
\item Assurer à nos collaborateurs un niveau d'expertise technologique
toujours de haut niveau.
\end{itemize}


\subsection{PARTIE 3: Organisations et méthodologies appliquées}
\begin{itemize}
\item Modéliser nos développements en utilisant le standard UML.
\item Utiliser des méthodes de développement objet (Java) et
d'application Web (XML).
\item Normaliser le code source développé.
\item Gérer le code source, les versions et les tâches avec les outils
disponibles.
\item Mettre en place des tests fonctionnels, des tests unitaires et
des tests d'intégration pour assurer la qualité du projet.
\end{itemize}



\subsection{PARTIE 4: Prestation}
\begin{itemize}
\item Respecter le planning et contraintes des clients.
\item Procéder systématiquement aux validations intermédiaires.
\item Informer les clients de tout éventualités ou imprévus.
\item Livrer une solution en conformité avec le cahier des charges.
\end{itemize}
Pour les réalisations nous nous engageons sur les résultats, les
délais et les coûts.


\subsection{PARTIE 5: Mise en place d'un espace client dédié pour le
suivi des projets}

Afin d'améliorer les échanges et le suivi du projet, nous mettons à
disposition des clients un espace dédié permettant:
\begin{itemize}
\item Consulter tous les documents associés au projet: Cahier des
charges, cahier de spécifications, analyses...
\item Suivre l'avancée du projet par l'intermédiaire d'un planning.
\item Accéder aux différentes versions du projet en cours de
développement.
\end{itemize}


\subsection{PARTIE 6: Propriété intellectuelle}

Nous nous engageons à:
\begin{itemize}
\item Livrer le code source, les documents techniques (analyses,

compétence si besoin.
\item Ne pas limiter les droits d'adaptation ou de modification de la
solution.

On s'engage à conclure avec les clients un contrat de service qui
