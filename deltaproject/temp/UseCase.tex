{\bf Paquetage Identification}
	\begin{itemize}
	\item Enseignant
		\begin{itemize}
		\item Se loger :
			\begin{itemize}
			\item Pré-requis : Avoir un compte et un mot de passe
		\item Description : L'utilisateur clique sur le lien {\it S'identifier}.\\
			Apparait une boîte de dialogue. \\
			Il entre le login et le mot de passe.\\
			Il valide.
			\item Post-requis : L'utilisateur est identifié dans l'intranet.
			\end{itemize}
		\item Se deloger :
			\begin{itemize}
			\item Pré-requis : Etre identifié/logé. 
			\item Description : L'utilisateur clique sur {\it Quitter}.\\
			Apparait un message de confirmation.\\
			Il confirme.\\
			\item Post-requis : L'utilisateur se trouve dans la page d'accueil de l'intranet.
			\end{itemize}
		\item Modifier paramètre :
			\begin{itemize}
			\item Pré-requis : Etre identifié/logé. 
			\item Description : L'utilisateur clique sur {\it Modifier ses paramètres}.\\
			Il est redirigé vers une page de paramètres personnels.\\
			Dans le cas du mot de passe : il entre son ancien mot de passe et confirme deux fois le nouveau. 
			\item Post-requis : paramètre modifié.
			\end{itemize}
		\end{itemize}
	\item Administrateur
		\begin{itemize}
		\item Ajouter compte :
			\begin{itemize}
			\item Pré-requis : Etre identifié/logé. 
			\item Description : L'administrateur clique sur le lien {\it Ajout de compte}.\\
			Il entre une page d'ajout de compte.\\
			Il ajoute un compte.\\
			Il ajoute un mot de passe qu'il confirme une seconde fois.\\
			Il aura la possibilité de générer un login et mot de passe automatiquement (le login est généré à partir du nom et prénom de l'utilisateur. Le mot de passe est généré aléatoirement).
			
			\item Post-requis : Nouveau compte ajouté dans la base de données.
			\end{itemize}

		\item Modifier compte :
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié. 
			\item Description : L'administrateur clique sur le lien {\it Modifier compte}.\\
			Il entre dans la page de modification de compte.\\
			Il recherche le compte soit par nom soit par login.\\ 
			Il a la possibilité de modifier les paramètres du compte.\\
			\item Post-requis : Le compte est modifié dans la base. 
			\end{itemize}

		\item Consulter compte :
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié. 
			\item Description : L'administrateur clique sur le lien {\it Consulter comptes}.\\
			Il entre dans la page de consultation des comptes.\\
			Il recherche le compte soit par nom soit par login.\\ 
			Il visualise le compte sélectionné.	
			\item Post-requis : Il obtient la page de consultation.\\
			La base de donnée reste inchangé.
			\end{itemize}

		\item Supprimer compte :
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié.
			\item Description : L'administrateur clique sur le lien {\it Supprimer comptes}.\\
			Il entre dans la page de suppression de comptes.\\
			Il recherche le compte soit par nom soit par login.\\
			Il sélectionne un compte.\\
			Il valide sa suppression. 
			\item Post-requis : Suppression d'un compte dans la base de donnée
			\end{itemize}
		\end{itemize}
	\end{itemize}

{\bf Paquetage Gestion Exercice}
	\begin{itemize}
	\item Enseignant
		\begin{itemize}
		\item Créer un exercice : 
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié.
			\item Description : L'utilisateur choisit un enseignement.\\
			Il utilise l'éditeur d'exercice de l'intranet pour créer un exercice.\\
			Il choisit son emplacement dans la hierarchie.\\
			Il choisit les mots clefs pour l'indexation de cet exercice.
			\item Post-requis : La base des exercices contient un exercice de plus. 
			\end{itemize}

		\item Importer exercice : 
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié.
			\item Description : L'utilisateur choisit un enseignement.\\
			L'utilisateur upload un exercice.\\
			Il choisit son emplacement dans la hierarchie.\\
			Il choisit les mots clefs pour l'indxation de cet exercice.
			\item Post-requis : La base des exercices contient un exercice de plus. 
			\end{itemize}

		\item Modifier exercice :
		 	\begin{itemize}
			\item Pré-requis : Etre logé/Identifié.\\
			L'exercice que l'utilisateur veut modifier existe.
			\item Description : L'utilisateur choisit un exercice.\\
			Il clique l'option {\it Modification}\\
			Il choisit le nouvel exercice 
			\item Post-requis : 
			\end{itemize}

		\item Supprimer exercice :
			\begin{itemize}
			\item Pré-requis : Etre logé/identifié.\\
			L'exercice que l'utilisateur veut modifier existe.	
			\item Description : L'utilisateur choisit un exercice.\\
			Il clique l'option {\it Suppression}\\
			Une fenêtre de confirmation apparait.\\
			L'utilisateur confirme son choix.\\
			L'exercice disparait des Tds auquels il appartenait et est déplacé dans le repertoire d'archives.
			\item Post-requis : L'exercice est supprimé des Tds et des projets et de la base des exercices.
			\end{itemize}

		\item Consulter un exercice :
			\begin{itemize}
			\item Pré-requis : Etre logé 
			\item Description : L'utilisateur choisit un enseignement puis l'exercice de son choix.\\
			Il peut consulter l'exercice et les corrections existantes.
			\item Post-requis : L'utilisateur visualise l'exercice choisit.
			\end{itemize}

		\item lister exercices :
			\begin{itemize}
			\item Pré-requis : Etre logé
			\item Description : L'utilisateur choisit un enseignement\\
			L'utilisateur demande le listing des exercices par un simple clique.
			\item Post-requis : La liste des exercices pour un enseignement donné est affiché.
			\end{itemize}
		\end{itemize}


		\item Rechercher exercice :
			\begin{itemize}
			\item Pré-requis : Etre logé
			\item Description : L'utilisateur rentre les mots clefs dans un moteur de recherche puis valide son choix.\\
			Une fenêtre affiche tous les exercices qui contiennent les mots clefs insérés.\\
			L'utilisateur peut cliquer sur une réponse pour afficher l'exercice.
			\item Post-requis : L'utilisateur peut choisir parmis les réponses de la recherche.
			\end{itemize}
	\end{itemize}

			Il clique pour le visualiser.
			\item Post-requis :Le devoir sélectionné est affiché au format HTML.
			\end{itemize}
				
		\item Rechercher un devoir :
			\begin{itemize}
			\item Pré-requis :Il faut qu'il y ait au moins un devoir de créé

		\end{itemize}

	\begin{itemize}
	\item Enseignant :
		\begin{itemize}
		\item : Créer un devoir :
			\begin{itemize}
			\item Pré-requis : Etre identifié.\\
			Il faut qu'au moins un exercice de type projet (gros exercice) soit enregistré.
			\item : Description : L'enseignant pénêtre dans l'intranet.\\
			Il s'identifie avec son login et son mot de passe.\\
			Il sélectionne le lien lui permettant de se diriger vers la page de création de projet.\\
			Il sélection l'enseignement pour lequel le projet est destiné.\\
			Ensuite, soit il sélectionne dans une liste l'exercice de type projet pré-enregistré souhaité pour ce projet ou soit l'"updload" à partir d'un fichier XML.\\
			Il valide sa sélection.
			\item Post-requis : Un nouveau projet est enregistré dans la liste des projets disponibles.
			\end{itemize}

		\item : Modification devoir :
			\begin{itemize}
			\item Pré-requis : Etre identifié.\\
			Il faut qu'au moins un projet soit enregistré.
			\item Description : L'enseignant pénêtre dans l'intranet. Il s'identifie avec son login et son mot de passe.\\
			Il sélectionne le lien lui permettant de se diriger vers la page des projets dans un enseignement.\\
			Il peut déplacer le projet dans un autre enseignement. \\
			Il peut modifier le projet par un autre exercice de type projet dans la liste des exercices.\\
			Il valide sa selection.
			\item : Avertissement : Si l'enseignant qui modifie le projet n'est pas son créateur, alors une sauvegarde de l'élément modifié sera effectué pour avoir la possibilité de revenir sur cette modification.
			\item Post-requis : Le projet est modifié dans la liste des projets disponibles.
			\end{itemize}

		\item : Suppression devoir :
			\begin{itemize}
			\item Pré-requis : Etre identifié.\\
			Il faut qu'au moins un projet soit enregistré.\\
			Il doit être le créateur du projet.
			\item : Description : L'enseignant pénêtre dans l'intranet.\\
			Il s'identifie avec son login et son mot de passe.\\
			Il sélectionne successivement les liens lui permettant de se diriger vers la page de modification de projet.\\
			Il peut demander la suppression du projet. Il valide sa demande.
			\item : Avertissement : Un enseignant ne peut pas supprimer un projet ne lui appartenant pas.
			\item : Post-requis : le projet est supprimé (s'il est son créateur).
			\end{itemize}

		\end{itemize}
	\end{itemize}

\section*{ Glossaire}

{\bf Administrateur} : Personne qui gère tous les comptes des enseigments.\\
{\bf EnseignantAdmin}: Enseignant qui rajoute et supprime des enseignements.\\
{\bf Enseignant}: Personne qui gère les exercices et les projets.\\
{\bf Consultant}: Personne qui visualise ou upload les TDs ou projet.\\
{\bf Exercice } il est formé d'un seul exercice.\\
{\bf TD}: il est formé d'un ensemble d'éxercices.\\
{\bf Projet}: il est formé d'un seul exercice.(souvent assez gros).\\
{\bf Enseignement}: un nouvel enseignement est constitué de la matière, l'année et la section.\\
{\bf Intranet } : Réseau accessible de l'interieur delafaculté.\\
{\bf se loger } : S'identifier par rapport au serveur.\\
{\bf Base de données} : Système de stockage d'information.\\
{\bf compte} : Association d'unlogin avec un mot de passe.\\
{\bf FAQ } : Fichier contenant les questions les plus fréquemmant posé et leurs réponses.\\
{\bf Upload } : Envoie d'un fichier vers le serveur
